# Chat2DB 全栈功能开发路线图 (Full-Stack Feature Development Roadmap)

本文档旨在提供一个从 0 到 1 的、以功能为导向的全栈开发计划。我们将遵循“最小可用 -> 逐步增强”的原则，让你在每个阶段都有明确的目标和可交付的成果。

### 核心开发理念

我们将采用“**纵向切片**”的方式进行开发。每个功能点，我们都同时考虑**后端（Rust）**和**前端（React）**的实现，确保完成一个，就能体验一个完整的闭环。

---

### Milestone 1: “数据库连接与原始查询” (基础核心)

**目标**：搭建应用的骨架，跑通最核心的数据库交互流程。

- **Feature 1.1: 实现单一数据库连接**
    - `[后端 Rust]`：创建一个 `connect_database` 命令。接收连接参数（主机、端口、用户、密码），使用 `sqlx` 建立与数据库（首选 PostgreSQL 或 MySQL）的连接。返回成功或失败的结果。
    - `[前端 React]`：构建一个最简的连接信息输入表单。点击“连接”时，调用后端的 `connect_database` 命令，并根据返回结果显示一个全局通知（如 "连接成功"）。

- **Feature 1.2: 执行并显示原始 SQL 查询结果**
    - `[后端 Rust]`：创建一个 `execute_query` 命令。它接收一个 SQL 字符串，执行它，并将查询结果（包含列信息和所有行数据）序列化成 JSON 格式返回给前端。
    - `[前端 React]`：创建一个包含一个大文本框（SQL 编辑器）和“执行”按钮的页面。点击按钮后调用 `execute_query` 命令，并将返回的 JSON 数据粗暴地显示在一个 `<pre>` 标签里，用于验证。

**此阶段成果**：一个能连接数据库、执行 SQL 并看到原始结果的、最基础的应用。

---

### Milestone 2: “可用性飞跃：成为真正的 SQL 客户端”

**目标**：加入核心 UI 元素，让应用从“能用”变成“好用”。

- **Feature 2.1: 连接管理与持久化**
    - `[后端 Rust]`：引入 `sqlx-sqlite`，在本地创建一个 `app.db` 文件。设计数据表用于存储用户保存的数据库连接配置。创建 `save_connection`, `list_connections`, `delete_connection` 等命令。同时，设计一个全局状态来管理当前所有活跃的数据库连接池。
    - `[前端 React]`：创建一个侧边栏，用于显示已保存的连接列表。允许用户点击列表中的项来快速连接，并提供新建、编辑、删除连接的 UI。

- **Feature 2.2: 数据库对象树状浏览器**
    - `[后端 Rust]`：创建 `get_database_metadata` 命令。该命令能根据一个已建立的连接，查询并返回其下的所有数据库(Databases)、模式(Schemas)、表(Tables)、视图(Views)等对象的层级列表。
    - `[前端 React]`：实现经典的三栏布局（左：连接与对象树，中：查询编辑器，右：信息面板）。左侧的对象树组件在连接成功后，调用 `get_database_metadata` 并动态渲染出可展开的树状结构。

- **Feature 2.3: 查询结果表格化**
    - `[前端 React]`：用一个功能强大的数据网格/表格组件（如 Ant Design 的 `Table`）替换掉之前简陋的 `<pre>` 标签。根据后端返回的列信息动态生成表头，并填充数据行。实现客户端的分页、排序等基础功能。

- **Feature 2.4: 多 Tab SQL 编辑器**
    - `[前端 React]`：将中部的查询编辑器改造成支持多标签页的组件。每个标签页维护自己独立的 SQL 内容、查询结果和历史。这是纯前端的 UI/状态管理工作。

**此阶段成果**：一个具备主流 SQL 客户端核心外观和操作体验的应用。

---

### Milestone 3: “效率提升：打造高效工作流”

**目标**：增加辅助功能，帮助用户更高效地工作。

- **Feature 3.1: SQL 查询历史记录**
    - `[后端 Rust]`：在每次 `execute_query` 成功后，将该 SQL、执行时间、状态等信息存入本地 SQLite 的历史记录表中。创建 `get_query_history` 命令以供前端查询。
    - `[前端 React]`：在右侧信息面板或独立的 Tab 中，创建一个“历史记录”列表，展示最近执行的 SQL，并支持点击历史记录来恢复 SQL 到编辑器中。

- **Feature 3.2: 保存和管理常用查询 (Snippets)**
    - `[后端 Rust]`：在 SQLite 中创建“已保存查询”表。创建增删改查 `(CRUD)` 的系列命令，用于管理用户自定义的查询片段。
    - `[前端 React]`：提供一个“收藏”或“保存”按钮，允许用户给当前编辑器里的 SQL 命名并保存。同时提供一个管理界面来搜索、使用、删除这些已存的查询。

- **Feature 3.3: 查看表结构 (DDL)**
    - `[后端 Rust]`：创建 `get_table_ddl` 命令。这需要编写针对不同数据库类型（如 MySQL 的 `SHOW CREATE TABLE ...`）的特定 SQL 查询。
    - `[前端 React]`：在左侧对象树的表名上增加右键菜单，提供“查看创建语句”选项。点击后，调用 `get_table_ddl` 命令，并在一个模态框或新 Tab 中展示表的 DDL 源码。

**此阶段成果**：一个功能完备、能显著提升日常 SQL 工作效率的工具。

---

### Milestone 4: “智能化：集成 AI 副驾”

**目标**：集成大语言模型，实现 Chat2DB 的核心亮点功能。

- **Feature 4.1: 自然语言转 SQL (NL to SQL)**
    - `[后端 Rust]`：使用 `reqwest` crate 编写一个服务，用于调用 OpenAI 或其他大模型的 API。创建 `natural_language_to_sql` 命令，它接收用户的自然语言描述，（可选）附加上下文（如表结构 DDL），然后向 AI 请求生成 SQL。需要安全地管理 API Key。
    - `[前端 React]`：在 SQL 编辑器界面中，增加一个 AI 对话面板或输入框。用户输入问题后，调用 `natural_language_to_sql` 命令，并将返回的 SQL 字符串填入编辑器中。

---

### Milestone 5: “产品化：设置、打包与完善”

**目标**：完成收尾工作，让应用成为一个可以公开发布的完整产品。

- **Feature 5.1: 应用设置**
    - `[后端 Rust]`：创建 `get_settings` 和 `save_settings` 命令，用于管理应用的配置项（如主题颜色、字体大小、AI Key 等），并将其保存在本地文件中。
    - `[前端 React]`：创建一个独立的“设置”页面。
- **Feature 5.2: 打包与分发**
    - `[全栈]`：执行 `yarn tauri build` 命令。学习如何替换应用图标，以及为 Windows 和 macOS 应用进行代码签名，以避免用户安装时出现安全警告。

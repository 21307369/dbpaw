## 多模型数据库接入架构方案（Oracle / MongoDB / Redis）

### Summary
基于当前仓库现状（单一 `DatabaseDriver`、SQL 优先命令面、关系型 `ConnectionForm` 与元数据模型），建议采用：
1. **Oracle 先接入**：沿用现有 SQL 架构，低风险扩展。
2. **MongoDB/Redis 后接入**：引入“多模型能力层”，避免把 NoSQL 强行塞进 SQL 抽象。
3. 产品形态采用你确认的方向：**多模型分层 + 只读优先**。

### Implementation Changes
1. 架构分层重构（先定义，后逐步迁移）
- 新增 `DataSourceAdapter` 总入口（仅做路由与能力声明），下分两类能力面：
  - `SqlAdapter`（Oracle/MySQL/Postgres/…）
  - `DocumentAdapter`（MongoDB）
  - `KeyValueAdapter`（Redis）
- 现有 `DatabaseDriver` 保留并收敛为 `SqlAdapter` 内部接口，不再承载 NoSQL 语义。
- 命令层改为“能力驱动”而非“SQL 驱动”：前端按能力调用对应命令，不共享不适用接口。

2. 能力声明机制（核心）
- 每个数据源实现统一 `capabilities` 描述（如：`query/sql`、`browse/collections`、`browse/keys`、`metadata/table_ddl`、`data/pagination`）。
- UI 通过 capabilities 决定显示：SQL Editor、集合浏览器、Key Explorer、DDL 面板等。
- 禁止“空实现”式兼容（例如 Mongo 返回伪表结构）以避免后续技术债。

3. 连接配置模型升级
- 连接配置从“固定关系型字段”升级为：
  - 通用字段（name、driver、network、auth、ssh、tls）
  - 驱动私有字段 `extra`（JSON）
- 本地存储兼容策略：保留现字段读取，新增 `extra` 承载 Oracle/Mongo/Redis 特有参数，旧连接无感迁移。

4. 三类数据库介入方式
- Oracle：进入 `SqlAdapter` 路线，复用现有查询/元数据/编辑器链路。
- MongoDB：进入 `DocumentAdapter`，首期只读能力固定为“连接测试 + DB/Collection 浏览 + 文档查询 + 分页查看 + 导出”。
- Redis：进入 `KeyValueAdapter`，首期只读能力固定为“连接测试 + DB/Key 浏览 + 值查看（按类型）+ 基础搜索 + 导出”。

5. 路线与难度评估（架构层）
- Oracle：**中等**。主要是方言和驱动适配，架构改动最小。
- MongoDB：**高**。数据模型与 SQL 抽象差异大，需要新能力面与新 UI 工作流。
- Redis：**中高**。命令与数据类型多样，查询模型与表格模型不一致，需要独立浏览交互。

### Public APIs / Interfaces / Types
1. 后端接口层
- 现有 SQL 命令保持不变（兼容已有数据库）。
- 新增 NoSQL 专用命令命名空间（例如：`document_*`、`kv_*`），不复用 `execute_query/get_table_*`。
- 新增 `get_connection_capabilities(connectionId)`，作为前端入口判定依据。

2. 前端类型层
- `Driver` 扩展：加入 `oracle | mongodb | redis`。
- `ConnectionForm` 变为“通用字段 + optional extra”模型。
- 查询结果类型拆分：
  - `SqlQueryResult`
  - `DocumentQueryResult`
  - `KeyValueQueryResult`

3. UI 路由层
- 连接后进入“能力面板路由”，不再默认直进 SQL Editor。
- SQL Editor 仅绑定 SQL-capable 数据源。

### Test Plan
1. 架构与兼容性
- 旧驱动（MySQL/Postgres/SQLite/MSSQL/ClickHouse/TiDB/MariaDB）行为不回归。
- 旧连接配置可读取，新增 `extra` 后可稳定序列化/反序列化。

2. 能力路由
- 不同驱动返回正确 capabilities。
- 前端按 capabilities 显隐入口，禁止出现不可用按钮。

3. 首期验收场景（只读）
- Oracle：连接、列库表、查询、分页、元数据查看可用。
- MongoDB：连接、库集合浏览、文档查询与分页可用。
- Redis：连接、DB/Key 浏览、按类型查看值可用。

4. 错误模型
- 网络/鉴权/超时错误在三类适配器下可被统一包装并展示。
- 不支持能力返回明确 `NOT_SUPPORTED`，而非通用失败。

### Assumptions
1. 已锁定产品方向：**多模型分层**，不做“Mongo/Redis SQL 化”。
2. 首期范围：**只读优先**，写入与管理能力后置。
3. 交付顺序：**Oracle 先，MongoDB/Redis 后**。
4. 本次规划不涉及具体驱动库选型与代码实现细节，仅定义可持续扩展的架构边界。

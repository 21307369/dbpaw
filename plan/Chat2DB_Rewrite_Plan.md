# Chat2DB Rust/Tauri 重写计划与学习指南

本文档旨在为你使用 Rust 和 Tauri 重写 Chat2DB 核心功能提供一个清晰的路线图，并就如何在你对技术栈不熟悉的情况下，高效地与 AI 协作，提供具体的建议。

---

## 第一部分：核心功能重写计划 (Roadmap)

我们将项目分解为多个阶段，每个阶段都有明确的目标和产出，方便你一步一步在 AI 辅助下完成。

### 阶段 0: 环境搭建与技术热身 (预计 1-2天)

**目标**：安装好所有开发工具，并成功运行一个最简单的 "Hello World" 项目，确保开发环境无误。

- [ ] **任务1: 安装基础环境**
  - 安装 [Rust](https://www.rust-lang.org/tools/install)。
  - 安装 [Node.js](https://nodejs.org/en) (推荐 LTS 版本) 和 `yarn` 包管理器。
  - 安装 VS Code 及关键插件: `rust-analyzer`, `Tauri`, `Prettier`。
- [ ] **任务2: 创建并运行 Tauri 初始项目**
  - 在终端运行 `yarn create tauri-app`，并按提示选择 `React` 和 `TypeScript` 模板。
  - 进入新创建的项目目录，运行 `yarn tauri dev`，确保能看到一个可以正常交互的桌面窗口。
- [ ] **任务3: 独立学习 `sqlx`**
  - 创建一个独立的、非 Tauri 的 Rust 小项目 (`cargo new sqlx_test`)。
  - 参照 `sqlx` 的官方文档，尝试连接到你本地安装的 PostgreSQL 或 MySQL 数据库，并成功执行一条 `SELECT 1;`。这个步骤有助于你隔离问题，提前掌握数据库连接的关键技术。

### 阶段 1: 实现最简核心闭环 (预计 1-2周)

**目标**：打通“连接数据库 -> 执行SQL -> 显示结果”的完整流程，验证技术可行性。

- [ ] **任务1: [前端] 创建连接界面**
  - 参考原 `chat2db-client` 项目，构建一个用于输入数据库连接信息（主机、端口、用户、密码）的表单界面。
- [ ] **任务2: [后端] 创建连接命令**
  - 在 Rust 代码中，创建一个 Tauri command，例如 `async fn connect_database(params: ConnectionParams) -> Result<String, String>`。
  - 在此函数内，使用 `sqlx` 尝试连接数据库。根据结果返回 `Ok("连接成功".into())` 或 `Err(error_message)`。
- [ ] **任务3: [前端] 调用连接命令**
  - 在连接表单的“连接”按钮点击事件中，使用 Tauri 的 `invoke` API 调用后端的 `connect_database` 命令，并通过 `alert` 或 `console.log` 显示返回结果。
- [ ] **任务4: [后端] 创建查询命令**
  - 创建另一个 command: `async fn execute_query(sql: String) -> Result<serde_json::Value, String>`。
  - 此函数接收 SQL 字符串，用 `sqlx` 执行，并将查询结果序列化为 JSON 格式返回。
- [ ] **任务5: [前端] 创建简易查询界面**
  - 创建一个包含文本输入框（用于写SQL）和“执行”按钮的简单界面。
  - 点击按钮时，调用 `execute_query` 命令，并将返回的原始 JSON 数据直接显示在一个 `<pre>` 标签内。

**里程碑**: **此阶段完成后，你将拥有一个可以实际连接数据库并执行查询的应用。这是整个项目最关键的验证点。**

### 阶段 2: 完善核心交互体验 (预计 2-3周)

**目标**：从“能用”进化到“好用”，构建核心的交互界面。

- [ ] **任务1: [后端] 多连接管理**
  - 设计一个 Rust 结构来安全地管理多个数据库连接池，可以研究 `std::collections::HashMap` 配合 `tokio::sync::Mutex`，并将其注册为 Tauri 的 `State`。
- [ ] **任务2: [后端] 获取数据库元数据**
  - 创建新的 command，用于获取数据库列表、指定数据库下的表 (Table) 和视图 (View) 列表。这需要执行特定数据库的元数据查询 SQL。
- [ ] **任务3: [前端] 构建左侧树状导航**
  - 参考原项目 UI，构建一个可展开的树状组件，用于展示已连接的数据库和其下的对象。
- [ ] **任务4: [前端] 美化查询结果**
  - 引入功能强大的表格组件（如 Ant Design 的 `Table`），将查询结果动态渲染成美观、可交互的表格。
- [ ] **任务5: [前端/后端] 多 Tab 查询窗口**
  - 实现多标签页功能，允许用户同时进行多个查询会话。

### 阶段 3: 扩展功能与本地持久化 (预计 1-2周)

**目标**：添加查询历史、保存查询等高频实用功能。

- [ ] **任务1: [后端] 集成 SQLite**
  - 使用 `sqlx` 和 `sqlx-sqlite` feature，为应用增加一个本地数据库 `app.db`，用于存储用户数据。
- [ ] **任务2: [后端] 创建本地存储命令**
  - 编写操作 SQLite 的 command，用于“保存查询”、“读取已保存列表”、“记录查询历史”等。
- [ ] **任务3: [前端] 实现相应功能界面**
  - 创建用于展示和管理已保存查询及历史记录的用户界面。

### 阶段 4: AI 集成与打包发布 (预计 1周)

- [ ] **任务1: [后端] 集成 AI 调用**
  - 使用 `reqwest` crate (Rust HTTP客户端) 来调用大模型 API（如 OpenAI）。
  - 创建一个 `natural_language_to_sql` command。
- [ ] **任务2: [前端] 创建 AI 功能 UI**
  - 添加输入框和按钮，让用户可以通过自然语言生成 SQL。
- [ ] **任务3: [打包] 构建与发布**
  - 学习并执行 `yarn tauri build` 命令，生成跨平台的安装包。
  - 学习替换应用图标、进行代码签名等发布相关流程。

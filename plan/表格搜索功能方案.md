### TableView 关键词搜索 + 回车跳转可行性与实施方案

#### Summary
在当前实现下可行，且可只改 `TableView` 前端逻辑完成，不依赖后端接口。  
按你已确认的方向：

- 搜索范围：`当前已加载 data`
- 回车行为：`首次定位，第 2 次起跳下一个`
- 入口位置：`hideHeader=true` 时使用 `表格右上角悬浮按钮`

核心结果是两层能力同时具备：
1. 关键词过滤：只显示包含关键字的行。  
2. 命中导航：在过滤结果内按回车循环跳到下一个命中单元格并自动滚动定位。

#### 详细实现设计（决策已定）

1. **新增搜索状态**
- `isSearchOpen: boolean` 控制弹层显示。
- `searchKeyword: string` 当前输入关键词。
- `searchCursor: { row: number; col: string } | null` 当前命中位置（用于“下一个”）。
- `matchedRows: number[]` 命中行索引（基于 `sortedData`）。
- `matchedCellsByRow: Map<number, string[]>` 每行命中的列名列表（用于定位和高亮）。

2. **匹配规则**
- 遍历 `sortedData`（不是 `currentData`），把每个单元格转字符串后做 `toLowerCase().includes(keyword)`。
- `null/undefined` 视为空，不命中。
- 空关键词时恢复原表（不做过滤，不保留 cursor）。

3. **过滤渲染**
- 新增 `displayData`：
  - 有关键词：`sortedData` 中仅保留命中行。
  - 无关键词：沿用原逻辑。
- 分页策略（按你选的“已加载数据范围”）：
  - 若外部分页（`onPageChange` 存在）：仅在当前已加载页数据里过滤。
  - 若本地分页：先过滤再分页，保证“结果集分页”体验一致。

4. **回车跳转（核心）**
- 在搜索输入框 `Enter`：
  - 若关键词刚变化或无 cursor：跳到第一个命中。
  - 否则跳到下一个命中，末尾后回到第一个（循环）。
- 跳转动作：
  - 更新 `selectedCell` 到命中单元格。
  - 若当前在编辑态，先 `commitEdit()` 再跳转，避免状态冲突。
  - 通过 cell ref 或 `querySelector` + `scrollIntoView({ block: "nearest" })` 保证可见。
- `Escape` 在搜索框内：关闭弹层，不影响已输入关键词（除非你后续要改为清空）。

5. **UI 入口与弹层**
- 在表格区域右上角加悬浮搜索按钮（`Search` icon + tooltip）。
- 点击后弹出小型输入框（popover 风格），自动 focus。
- 弹层中展示计数：`x rows / y cells matched`，无结果提示 `No matches`。

6. **视觉反馈**
- 命中单元格加轻量高亮（背景色）。
- 当前 cursor 命中单元格加更强高亮（边框/环）。
- 保持与现有 `selectedCell` 样式兼容，避免覆盖编辑态边框。

7. **键盘冲突规避**
- 仅在搜索输入框聚焦时拦截 `Enter/Escape`。
- 不修改现有全局热键（`Cmd/Ctrl+S`, `Esc` 撤销）行为边界。

#### 测试用例与验收标准

1. **基础过滤**
- 输入关键字后，仅显示包含关键字的行。
- 清空关键字后恢复完整数据。

2. **回车导航**
- 首次回车定位第一个命中。
- 连续回车按顺序跳转，末尾后循环到首个。
- 无命中时回车不报错，显示 `No matches`。

3. **编辑态兼容**
- 正在编辑单元格时触发回车导航，不丢失编辑提交/取消逻辑，不报错。

4. **分页与排序兼容**
- 本地排序后搜索结果仍按当前排序顺序导航。
- 外部分页场景只在当前已加载数据内生效，行为一致。

5. **hideHeader 场景**
- `hideHeader=true` 时仍可看到右上角悬浮搜索按钮并可完整使用。

---

#### Assumptions & Defaults
- 搜索大小写不敏感（`case-insensitive`）。
- 搜索范围限定为前端当前持有的 `data`，不跨后端未加载页。
- 关键词匹配语义为“包含”（`includes`），不引入正则/高级语法。
- 命中导航目标为“单元格级别”，并同步更新 `selectedCell`。

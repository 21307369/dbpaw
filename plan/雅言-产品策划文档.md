# 雅言 —— 产品策划文档

> 一个把现代口语翻译成古诗词、成语、文言文的装逼工具。
> 版本：V0.1 MVP
> 日期：2026-05-12

---

## 一、产品概述

### 1.1 一句话描述
**雅言**是一个把现代人的口语（尤其是网络用语、脏话、感叹词）一键翻译成古诗词、成语、文言文金句的轻量工具，让用户在聊天、发朋友圈、写评论时，用最少的输入获得最高的"文化逼格"。

### 1.2 核心口号
- **主口号**："奈何本人没文化？一句古诗走天下。"
- **副口号**：你的嘴替，来自唐朝。

### 1.3 解决的问题
- 想装逼但背不出诗
- 想吐槽但不想说脏话
- 想发朋友圈但想不出文案
- 吵架时想优雅地阴阳对方

### 1.4 不做的事情（边界）
- 不做完整的古诗词学习/教育功能（不是第二个"西窗烛"）
- 不做诗词创作/格律检测
- 不做长篇翻译（只聚焦**单句/短句**的"金句替换"）

## 二、目标用户

| 用户画像 | 典型场景 | 核心诉求 |
|---------|---------|---------|
| **社交活跃青年**（18-30岁） | 微信群聊天、微博评论 | 快速复制一句有逼格的回复，碾压全场 |
| **文案/运营人员** | 写朋友圈、小红书、公众号标题 | 找有韵味的短句做文案素材 |
| **"吵架想赢"党** | 网上对线、回怼 | 用文言文优雅地骂人，显得有文化 |
| **学生党** | 作文、情书、毕业留言 | 给文字加点古风韵味 |

### 2.1 用户痛点（直接引用）
> "每次想说'卧槽牛逼'的时候，看到别人回一句'此曲只应天上有'，就觉得自己像个文盲。"

> "想发朋友圈晒个晚霞，只会说'卧槽好美'，想找个有文化又不矫情的句子，搜半天找不到。"

> "吵架的时候对方骂脏话，我回一句文言文，气势上就已经赢了。"

## 三、产品形态与版本规划

### 3.1 第一版：网页版 MVP（V0.1）
**目标：验证需求，测试传播性**

- **形态**：响应式网页，手机端优先
- **核心功能**：输入框 → 选风格 → 生成3-5条金句 → 一键复制
- **开发周期**：1-2周
- **成本**：前端1人 + AI API调用费用（极低）

**为什么先上网页版：**
1. 零安装成本，链接即传播
2. 开发最快，无审核流程
3. 跨平台（手机、电脑、平板）
4. 方便接入微信/微博分享

### 3.2 第二版：微信小程序（V0.5）
**目标：承接社交传播，降低使用门槛**

- 在网页版验证有传播力后，移植到小程序
- 增加"分享到微信群"、"生成朋友圈卡片"功能
- 接入微信登录，保存历史记录

### 3.3 第三版：App/进阶网页（V1.0）
**目标：留存用户，建立内容壁垒**

- "神句库"：沉淀高质量的用户生成配对
- "每日一雅"：每日推送一句场景化古文
- "对战模式"：两个用户输入同一句话，比谁生成的更绝

### 4.3 功能模块详解

#### 模块A：核心翻译引擎
- **输入**：用户口语（1-50字）
- **风格标签**（单选，影响Prompt）：
  - 🔥 **豪迈**：李白式、气势磅礴
  - 🍃 **文艺**：婉约、有意境
  - 😏 **阴阳**：嘲讽、怼人、优雅地骂人
  - 💔 **伤感**：emo、失落、遗憾
  - 😂 **搞笑**：不正经、反差萌
  - 🎭 **正经**：官方、得体、适合正式场合
- **输出**：3-5条金句，每条包含：原文、作者、出处

#### 模块B：一键复制
- 复制纯文本（带/不带出处可选）
- 复制后Toast提示："已复制，去粘贴装逼吧"

#### 模块C：换一句
- 对某一条结果不满意，单独刷新该条
- 保留其他结果

#### 模块D：分享卡片生成
- 选择一条结果 → 生成精美图片卡片
- 卡片样式：
  - 宣纸纹理背景 + 书法字体
  - 底部带品牌Logo和二维码
  - 可选加"今日雅言"标签
- **这是传播裂变的核心功能**

#### 模块E：快捷入口（降低冷启动成本）
- **键盘侠急救包**：预置高频网络用语的翻译
  - "绝了" / "服了" / "笑死" / "离谱" / "蚌埠住了" / "yyds"
- **吵架优雅指南**：预置怼人场景的翻译
  - "你脑子有病" → "君有疾在腠理，不治将恐深"
  - "你算老几" → "尔曹身与名俱灭，不废江河万古流"
- **今日签**：随机生成一句诗作为每日运势

## 五、与 ChatGPT 的差异化（体验壁垒）

### 5.1 直接对比

| 体验维度 | 用 ChatGPT | 用雅言 |
|---------|-----------|--------|
| **输入成本** | 需要写提示词："请用古诗词表达..." | 直接打字，选风格，点生成 |
| **输出数量** | 通常给1-2个，不满意要重新问 | 一次给3-5个备选 |
| **输出格式** | 可能带解释、分析、废话 | **只给金句+出处**，干净 |
| **情绪细分** | 通用翻译，没有情绪标签 | **6种风格一键切换** |
| **使用场景** | 生成后手动复制，格式不定 | **一键复制+生成卡片** |
| **灵感获取** | 不知道问什么的时候就干瞪眼 | **快捷入口**直接给高频场景 |
| **社交传播** | 无 | **分享卡片带品牌露出** |
| **速度** | 打开ChatGPT网页/ App，登录... | 打开链接，3秒出结果 |

### 5.2 核心壁垒总结
1. **交互聚焦**：ChatGPT是通用工具，雅言是专用工具。专用意味着更少的选择、更快的路径。
2. **情绪映射**：ChatGPT不知道你想"豪迈地牛逼"还是"阴阳地牛逼"，雅言用标签解决。
3. **社交闭环**：ChatGPT不能帮你生成朋友圈图片，雅言可以。
4. **内容沉淀**：积累"神句库"后，用户可以发现"别人生成的绝妙配对"，这是社区价值。

## 六、技术方案（MVP）

### 6.1 架构

```
用户浏览器（React/Vue 单页应用）
    ↓
后端 API（Node.js/Python/Cloudflare Workers）
    ↓
AI 大模型 API（OpenAI / Claude / DeepSeek / 通义千问）
```

### 6.2 核心Prompt设计

```
你是一位精通中国古典文学的"雅言翻译官"。
你的任务是把现代人的口语翻译成最有韵味的古诗词、成语或文言文金句。

用户输入：{user_input}
用户选择的风格：{style_tag}

请严格按照以下规则输出：
1. 给出3个最佳选项
2. 每个选项必须是单句或对联形式，不超过30字
3. 每个选项必须标注出处（作者名《作品名》）
4. 风格必须严格匹配用户选择的标签：
   - 豪迈：李白、苏轼式的奔放气势
   - 文艺：婉约、意境优美
   - 阴阳：优雅地嘲讽、怼人
   - 伤感：凄凉、遗憾、emo
   - 搞笑：反差、不正经
   - 正经：得体、适合正式场合
5. 如果输入含有脏话，翻译成有力但不粗俗的古文表达
6. 如果输入过于现代找不到对应古诗，可用成语或四字词替代
7. 不要解释，不要分析，只输出结果

输出格式：
1. "诗句原文" ——作者《作品》
2. "诗句原文" ——作者《作品》
3. "诗句原文" ——作者《作品》
```

### 6.3 成本估算
- 单次调用约 500-1000 tokens
- 按 GPT-4o-mini / DeepSeek-V3 计算，单次成本约 ¥0.01-0.05
- 日活1000用户，人均10次，日成本约 ¥100-500

### 6.4 前端技术栈建议
- **框架**：React + Tailwind CSS（快速响应式）
- **分享卡片**：html2canvas 或 dom-to-image（DOM转图片）
- **字体**：引入书法字体（如"站酷庆科黄油体"、"思源宋体"）

## 八、风险与应对

| 风险 | 应对方案 |
|-----|---------|
| AI生成错误出处/张冠李戴 | Prompt加约束 + 后端做常见名句的白名单校验 |
| 内容被用于恶意攻击 | 加敏感词过滤，阴阳风格控制在"优雅嘲讽"而非"人身攻击" |
| 用户新鲜感消退快 | 持续更新"快捷入口"场景，蹭热点做限时主题 |
| 被ChatGPT/大模型厂商抄功能 | 快速积累内容壁垒（神句库、社区）、建立品牌认知 |
